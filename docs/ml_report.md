# ML 리포트 (v2) — 규칙 vs ML(7주차) + 발음형 인식기 미세조정(후속)

이 리포트는 두 국면을 담는다.
- **Part 1 — 7주차: 규칙 기반 vs ML** (아래 §1~7). 실사용 98건으로 학습한 오류 예측 모델이
  문장 추천에서 규칙 베이스라인을 **precision@5 0.4 → 0.6** 으로 앞섰고, 합성 분석에서

  **오탐을 실질 52% → 6%로** 낮췄다.

> 상세 3부작: [+1주 데이터 정제](l2_error_distribution.md) · 실측 종합

> 미세조정 결과 [finetune_results.md](../experiments/results/finetune_results.md).

**① 가정의 반증 (실데이터 측정)** — AI Hub "교육용 외국인 한국어 음성" 실발화 2만 건으로
현 시스템을 측정하니 전제가 두 번 뒤집혔다.

| | 합성 가정(7주차) | 실데이터(후속) |
|---|---|---|
| 받침 감지율 | 35% (최약점이라 판단) | **73%** (합성 35%는 TTS 한계였음) |
| 진짜 병목 | — | **정발음 오탐률 52%** (정확히 발음한 발화의 절반에 헛오류) |
| 오탐의 원인 | — | **83%가 Whisper 오인식** (짧은 L2 발화를 다른 단어로 환각) |

**② 표적 재정의 + 실행** — "받침 recall 개선"이 아니라 **"Whisper→발음형 음소 인식기 교체로
오탐 제거"**. 학습 라벨 = AI Hub phones(사람 전사 발음형), 화자 단위 분할(누출 0),
`kresnik/wav2vec2-large-xlsr-korean` + CTC 자모 헤드(39종) 미세조정.

**③ 결과 (before/after)**

| 지표 | Whisper (before) | wav2vec2 (after) |
|---|---|---|
| 발음형 CER | — | **3.9%** (5 epoch) |
| **정발음 오탐률(실질)** | **52%** | **6%** |
| 자모-검출 가능 분절음 오류 회수율 | — | **99%** |

- 오탐 6%는 거리≥1 플래그 20%를 phones 정본으로 분해한 값(순수 모델오탐 6% + 화자 실제 발음변이
  14%). Whisper 52% 대비 약 9배 감소.
- **정직한 한계**: 자모-사전형 비교의 감지 상한은 27%(사람이 지각한 분절음 오류의 다수가
  자모-표준형 편집거리로 안 드러남) — 프론트엔드가 아니라 비교 패러다임의 한계, 다음 과제.

**계보**: 1주차 표기 수렴 문제 발견 → 2주차 wav2vec2 비음화 감지 검증 → 7주차 합성 학습 실패의
교훈 → 후속 오탐 발견·표적 재정의 → 미세조정으로 해결. 가정을 실데이터로 뒤집는 **정직한 반증**이
이 프로젝트의 일관된 서사다.

## Part 1 — 규칙 기반 vs ML (7주차)

## 1. 데이터셋

| 구분 | 건수 | 출처 | 용도 |
|---|---|---|---|
| user | 98 | 브라우저 실사용 (본인 육성, 33문장 전체 커버) | **학습 + 평가** (5-fold CV) |
| synth | 198 | TTS 오발음 주입 (보이스 2종 × 33문장 × 정상1+변형2) | 감지율 분석 (과제 D) |

- 시도(attempt)마다 문장 발음형·유형, 자모 단위 오류 리포트(엔진 산출)가 라벨로 붙는다.
- 합성 데이터는 자모 혼동 쌍(ㄹ↔ㄴ, ㅓ↔ㅗ, 받침 탈락 등)을 무작위 주입해 만들었고,
  주입 내역이 `experiments/results/synth_labels.csv`에 기록되어 라벨이 정확하다.
- **특징 벡터(79차원)**: 발음형의 자모 구성(bag-of-jamo 68) + 구조 통계(음절 수, 받침 수,
  겹받침 수) + 발음 규칙 유형 one-hot(9). 비교가 발음형 공간에서 이뤄지므로 특징도 발음형에서 추출.

## 2. 과제 A — 오류 발생 예측 (이진 분류)

**질문**: 문장 특징만 보고 "이 문장에서 사용자가 틀릴까"를 맞힐 수 있는가?
(user 98건, 오류 25건 — 5-fold 층화 교차 검증)

| 모델 | CV ROC-AUC | CV F1 |
|---|---|---|
| 베이스라인 (다수 클래스) | 0.500 | 0.000 |
| **로지스틱 회귀 (채택)** | **0.652 ± 0.185** | 0.379 |
| 랜덤 포레스트 | 0.644 ± 0.183 | 0.391 |

- 무작위(0.5)보다 유의미하게 높지만 표준편차가 크다(±0.185) — **표본 98건의 한계**.
  fold마다 오류 시도가 5건 수준이라 분산이 클 수밖에 없다.
- 채택 모델(로지스틱 회귀)은 `data/models/error_clf.joblib`로 저장되어
  `GET /recommend`가 사용한다.

## 3. 과제 B — 추천 품질: 규칙 기반 vs ML (핵심 비교)

**질문**: "다음에 연습할 문장"을 고를 때 ML이 규칙 기반보다 나은가?

- 정답 정의: 사용자가 실제로 1회 이상 틀린 문장 (33문장 중 13개, 기저율 0.394)
- 모델 랭킹: **out-of-fold** 예측 확률의 문장별 평균 (in-sample 낙관 제거)
- 규칙 랭킹: 6주차 취약 자모 TOP 5를 많이 포함한 문장 우선 (규칙 기반 전신)

| 방법 | precision@5 | 기저율 대비 |
|---|---|---|
| 무작위 (기저율) | 0.394 | — |
| 규칙 기반 (취약 자모 TOP 5) | 0.400 | +2% |
| **ML (오류 확률 랭킹)** | **0.600** | **+52%** |

**해석**: 규칙 기반은 "자주 틀린 자모가 든 문장"을 고르지만, 그 자모가 들어 있어도
쉬운 문맥이면 안 틀린다. 모델은 자모 구성·문장 구조·규칙 유형을 함께 보므로
"실제로 틀릴 문장"을 더 잘 고른다. 단 33문장 중 상위 5개 비교라 표본이 작다 —
방향성은 확인됐고, 신뢰구간을 좁히려면 데이터가 더 필요하다.

**서비스 연결**: 연습 화면의 [🎯 추천 문장] 버튼 → `GET /recommend` →
`점수 = P(오류) × (1 + 0.5 × 취약자모 겹침)` 상위 문장 + 추천 이유 표시. (완료 기준 충족)

## 4. 과제 C — 부정적 결과: "합성으로 학습"은 왜 실패했나

처음 설계는 "합성 198건으로 학습 → 실사용 98건으로 평가"였다.
결과(오류 성분 4-분류): 베이스라인 macro-F1 0.213 vs 랜덤 포레스트 0.284 — 사실상 무의미.

**원인은 데이터가 아니라 설계다.** 합성 오류는 문장과 무관하게 **무작위로 주입**했으므로,
"어떤 성분을 틀릴지"가 문장 특징과 설계상 독립이다. 즉 모델이 배울 신호가 애초에 없다.
실제 사람의 오류는 문장 난이도에 종속되므로 실사용 데이터에는 신호가 있다(과제 A가 증명).

> 교훈: 합성 증강은 "라벨 분포를 통제한 평가·분석"에 쓰고, "사람 행동 예측 학습"에는
> 실데이터를 써야 한다. 이 구분이 이번 주의 가장 큰 배움이다.

## 5. 과제 D — 합성 데이터의 진짜 쓸모: 오류 감지율 지도

주입한 오류를 시스템(Whisper + 비교 엔진)이 얼마나 잡아내는지 — 합성 라벨이 정확하기에
가능한 분석이다 (변형 132건 대상).

| 주입 성분 | 감지 | 전체 | 감지율 |
|---|---|---|---|
| 초성 | 33 | 49 | **67%** |
| 중성 | 40 | 63 | **63%** |
| **받침** | 7 | 20 | **35%** ⚠️ |

가장 안 잡히는 혼동 쌍 (2회 이상 주입 기준):

| 혼동 | 감지/전체 | 추정 원인 |
|---|---|---|
| 중성 ㅡ→ㅜ | 0/5 | Whisper 교정 또는 TTS 음향 구분 미미 |
| 초성 ㄱ→ㅋ | 0/4 | 격음 구분이 문맥 교정에 흡수 |
| 받침 ㄹ 탈락 | 0/2 | 연음 규칙이 발음형 공간에서 차이를 흡수 |
| 받침 ㄴ→ㅇ | 0/2 | 비음 간 혼동 — Whisper가 단어로 복원 |

**시사점**: 받침 오류 감지율 35%는 1주차 결론("표기 수렴 문제")의 정량 버전이다.
받침 피드백의 신뢰도를 높이려면 음소 단위 인식(wav2vec2, 2주차 검증에서 비음화 감지 확인)을
받침 구간에 조준 적용하는 하이브리드가 다음 단계다.
단, 감지 실패에는 TTS가 해당 혼동을 음향적으로 뚜렷이 내지 못했을 가능성도 섞여 있다
(합성 기반 분석의 한계).

## 6. 한계와 다음 단계

- **표본 크기**: 사용자 1명, 98건. AUC 분산(±0.185)과 추천 비교(상위 5개)의 통계적 힘이 약하다.
  다중 사용자 데이터가 모이면 사용자별 모델·콜드 스타트 전략이 다음 과제.
- **합성 분포**: 주입 혼동 쌍이 실제 학습자(L2) 오류 분포와 다르다 — 감지율 분석에는 충분하나
  행동 모델 학습에는 부적합 (과제 C).
- **받침 감지 35%**: wav2vec2 조준 검사(2주차 검증 자산 재사용)로 보완 가능 — 8주차 이후 백로그.
