# 소리새김 — 후속 계획: ML 고도화 (v1)

> 8주 본 프로젝트 완료 후의 확장 로드맵. 목표는 두 가지다.
> **① 제품 지표**: 받침 오류 감지율 35% ([ml_report](ml_report.md) 과제 D)를 wav2vec2 미세조정 + 조준 검사로 끌어올린다.
> **② 경력 지표**: fine-tuning(post-training)과 데이터 정제를 "근거 있는 키워드"로 만든다.
> 주당 10~15시간 가정, 총 6주. 본 계획도 v1과 같은 원칙 — 밀리면 '선택'을 버리고 필수만 지킨다.

**전체 흐름 요약**

| 주차 | 테마 | 필수 산출물 |
|---|---|---|
| +1주 | 데이터 정제 파이프라인 | 품질 필터 스크립트 + 정제 리포트(제외 사유별 건수) |
| +2주 | 학습 데이터셋 구축 | 발음형 라벨 데이터셋(공개 음성 + 합성 v2) + 화자 분리 분할 |
| +3~4주 | wav2vec2 미세조정 | 발음형 CTC 모델 + 기준선 대비 지표 (CER, 받침 감지율) |
| +5주 | 하이브리드 통합 | 받침 조준 검사 API 통합 + before/after 지표 |
| +6주 | 문서화 + (선택) Whisper FT 검토 | ml_report v2 · README 갱신 · 이력서 문구 |

**이 계획이 성립하는 근거 (기존 자산 재사용)**

- 2주차 검증: `kresnik/wav2vec2-large-xlsr-korean`이 비음화 미적용을 감지함을 확인 (`experiments/wav2vec2_check.py`, 결과 md). 미세조정의 베이스 모델과 평가 스크립트가 이미 있다.
- 7주차 합성 파이프라인: `experiments/synth_data.py` — 틀린 텍스트를 TTS로 읽혀 **라벨이 정확한 오발음 음성**을 만드는 생성기. 학습 데이터 증폭에 그대로 쓴다.
- 감지율 측정 파이프라인: `synth_labels.csv` + jamo_confusion 분석 — **받침 35%가 공식 기준선(before)**.
- ⚠️ 과제 C의 교훈과 충돌하지 않는다: 합성이 실패한 건 "사람 행동 예측" 학습이었기 때문.
  이번 과제는 **음향→발음형 전사** 학습이라, 합성 음성의 라벨(실제로 발성된 내용)이 정확하므로 유효하다.
  이 구분을 리포트에 명시할 것 (면접 단골 질문 예상 지점).

## +1주차 — 데이터 정제 파이프라인 (≈12h)

"데이터 정제" 키워드를 실체로 만드는 주. 산출물은 코드가 아니라 **제외/보정 기준과 그 근거가 담긴 리포트**다.

### 작업

  - 문헌·공개 자료 조사: 국립국어원 한국어 학습자 말뭉치 오류 통계, L2 발음 오류 연구 (모어별 상위 혼동: 예 — 일본어권 받침 탈락·ㄹ/ㄴ, 중국어권 평음/격음/경음, 영어권 ㅡ/ㅜ).

## +2주차 — 학습 데이터셋 구축 (≈13h)

미세조정의 병목은 코드가 아니라 데이터다. **발음형(g2pk) 라벨**이 이 프로젝트 고유의 설계 포인트 —
"모델도 표기로 학습되어 표기 수렴에 갇힌다"(2주차 결론)를 라벨 공간 교체로 정면 돌파한다.

### 작업
- [ ] **공개 한국어 음성 확보** (3h): 1순위 **Zeroth-Korean** (~51h, CC-BY, 즉시 다운로드). 보조: KSS(단일 화자 12h).
  - AI Hub(KsponSpeech 등)는 승인 대기가 있으므로 **신청만 걸어두고 기다리지 않는다** (결정 포인트 참조).
- [ ] **발음형 라벨링 파이프라인** (4h): 표기 전사 → g2pk 발음형 → 자모 시퀀스. 실패/애매 케이스(숫자·영문·문장부호) 정규화 규칙 정의. g2pk 처리 불가 문장은 제외하고 사유 기록 (정제 리포트에 합산).
- [ ] **데이터셋 병합·분할** (3h): 공개 데이터(발음형 라벨) + 합성 v2(오발음, 라벨=주입된 발음) + 실녹음.
  - 분할 원칙: **화자 단위 분리** (합성은 보이스 단위) + **1주차 인간 녹음 20건은 전량 테스트 전용** (골드셋, 학습 금지).
  - 규모 목표: 학습 5~10h 분량이면 충분 (사전학습 모델의 CTC 헤드 적응이 목적이므로 수백 시간 불필요).
- [ ] **HF datasets 포맷 변환 + 로더 검증** (3h): Colab에서 스트리밍 가능한 형태(arrow/parquet)로 업로드 준비.

### 완료 기준
- train/val/test 매니페스트가 존재하고, 테스트셋에 학습 화자가 없음을 스크립트로 검증했다.

### 결정 포인트
| 상황 | 결정 |
|---|---|
| Zeroth 다운로드·라벨링 순조 | 계획대로 진행 |
| g2pk 라벨 오류율이 눈에 띔 (샘플 100문장 수동 검수 기준 >5%) | 라벨 신뢰 구간을 좁힌다 — 규칙 유형이 명확한 문장만 선별 사용 |
| AI Hub 승인 도착 | +3주차 이후 증분 학습 데이터로만 활용 (계획 지연 금지) |

## +3~4주차 — wav2vec2 미세조정 (≈24h)

핵심 주. **베이스**: `kresnik/wav2vec2-large-xlsr-korean` (한국어 음향 적응 완료 — 2주차 검증 자산).
**과업**: CTC 헤드를 발음형 자모 vocab으로 교체 후 미세조정. **환경**: Colab GPU (T4/L4), fp16, gradient accumulation.

### 작업
- [ ] **기준선 고정** (3h): 미세조정 전 모델로 테스트셋 평가 — 발음형 CER, 받침 감지율(합성 v2 held-out), 1주차 골드셋 20건. **이 숫자가 모든 비교의 before다.** (`ml/eval_phoneme.py`로 스크립트화 — 이후 반복 사용)
- [ ] **학습 코드** (6h): HF `Trainer` + `Wav2Vec2ForCTC`.
  - vocab: 자모 단위 (초/중/종성 토큰 — engine/hangul의 분해 로직 재사용), feature encoder(CNN) 동결.
  - 1차 하이퍼파라미터: lr 1e-4(헤드)/3e-5(트랜스포머), 5~10 epoch, early stopping(val CER).
- [ ] **1차 학습 + 오류 분석** (5h): val CER 곡선, 혼동 행렬(자모 단위) — 어떤 받침이 여전히 안 잡히는지 2주차 표와 대조.
- [ ] **2차 학습 (개선 1~2가지만)** (5h): 오류 분석 근거로 선택 — 예: 합성 v2 비중 조정, 받침 포함 샘플 오버샘플링, SpecAugment 강화. **한 번에 하나씩 바꾸고 기록** (ablation 표가 곧 이력서 근거).
- [ ] **최종 평가 + 모델 확정** (5h): 테스트셋·골드셋에서 before/after 표 작성. 모델 카드(학습 데이터 구성·지표·한계) 작성 후 저장.

### 완료 기준
- before/after 표가 있다: 발음형 CER, **받침 감지율 35% → X%** (합성 held-out), 골드셋 20건 정성 비교.
- 실패 시에도 완료로 친다 — 단, "왜 안 됐는지"의 오류 분석이 있어야 한다 (과제 C처럼 부정적 결과도 기록).

### 결정 포인트
| 결과 | 결정 |
|---|---|
| 받침 감지율 유의미 개선 (35% → 50%+) | +5주차 통합 진행 |
| CER은 개선됐지만 받침 감지율 정체 | 전역 성능이 아니라 조준 검사에 집중 — +5주차를 "조준 구간 한정 판정"으로 설계 (전역 오인식 노이즈 회피, 2주차 결론 재활용) |
| 학습 자체가 불안정 (CER 발산) | lr 축소 + 헤드만 학습으로 후퇴, 그래도 안 되면 XLS-R 300m에서 재출발 |

## +5주차 — 하이브리드 통합: 받침 조준 검사 (≈12h)

모델을 제품 지표로 연결하는 주. 전역 재점수화가 아니라 **"이 문장의 규칙 위치만 조준 검사"** (2주차 결론의 설계).

### 작업
- [ ] **조준 구간 산출** (3h): g2pk 변환 전후 diff로 문장 내 음운 규칙 적용 위치(받침·비음화 등) 추출 — engine에 유틸 추가.
- [ ] **조준 판정 로직** (4h): wav2vec2 발음형 출력에서 해당 구간만 정렬·판정, Whisper 경로 점수에 보조 신호로 결합 (가중 방식은 단순 시작: 받침 성분만 wav2vec2 판정 우선).
- [ ] **API·프론트 통합** (3h): 응답에 `jongseong_check` 필드 추가, 결과 화면 툴팁에 근거 표시. CPU 추론 지연(문장당 1~2초) 고려해 **옵션 플래그**로 시작.
- [ ] **엔드투엔드 검증** (2h): 골드셋 + 신규 실녹음 10건으로 실사용 흐름 확인, pytest 케이스 추가.

### 완료 기준
- 브라우저에서 받침 오류 문장을 읽으면 조준 검사 결과가 하이라이트에 반영된다.
- **시스템 수준 before/after**: 합성 v2 주입 오류에 대한 전체 파이프라인 감지율 재측정 (ml_report 과제 D 표의 v2).

| 도메인 문제 해결 | 표기 수렴 문제 → 발음형 라벨 공간으로 정면 돌파 (1주차 실험부터 이어지는 서사) |

## 리스크
